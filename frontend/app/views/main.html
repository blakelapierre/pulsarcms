Developing On Pulsar

Pulsar doesn't get themed. No CMS should. Pulsar does not want to be a rigid monolith that lets you edit a little CSS and add an HTML file. That whole paradigm is entirely too limiting to your designs. Instead, developers are encouraged to build a custom client/front-end of their own design using AngularJS (obviously my preference) or whatever else you want to use if that's what you really want to do. Building on AngularJS means you can use the ng-pulsar framework and finish faster.

Because Pulsar isn't a single server, it's two. The Pulsar API server is powered by Node.js atop MongoDB. Separately, the Pulsar front end isn't even technically a server. It's an HTML5 single-page application client that works more like desktop software than a web page. And, you have complete freedom in its hosting if you even decide to use it.

I am using Apache httpd. But, if "it" throws a file onto a wire when properly asked to do so, it'll work. Pulsar's front end really is just a bunch of static, cache-busted, concatenated, minified and crunched standard MIME-typed assets. I'm serving this very site from atop a mountain in Pennsylvania using ridiculously unstable DSL service with a 1Mbps upstream connection to you when it decides to stay connected. So, yes. It's "tested" (daily).
